# static userDataValidator(options, …fields) → {Array.<function()>}

Parameters:
Name Type Attributes Description
options Object.<string, string>

this param maps a models.User property to its name in the body. For example, you want to set a new password for the users, but this password is located under the newPassword property in the body. In this case you should pass {password: 'newPassword'} under the options param. Pass a falsy value if you don't need any conversion, described above
fields string <repeatable>

the fields that should be validated for correctness. See models.User for the list of all acceptable fields. Read-only fields (like _id or updated) cannot be used here
